## 📄 PRD: 오목 (Nexus-Omok-Game) 게임 (.NET 8.0 / WPF)

### 1\. 개요

  * **문서 목적:** 본 문서는 **Windows 데스크톱 환경**을 위한 .NET 8.0 및 WPF 기반의 2인용 로컬 오목(Gomoku) 게임 MVP(Minimum Viable Product) 버전 개발을 위한 제품 요구 사항을 정의합니다.
  * **제품명:** Nexus Omok Game
  * **제품 목표:** 사용자가 자신의 Windows PC에서 간단하고 직관적으로 2인(로컬) 오목 게임을 즐길 수 있도록 합니다.
  * **타겟 사용자:** 친구나 가족과 동일한 PC에서 빠르고 간단한 보드게임을 즐기려는 캐주얼 게이머.

-----

### 2\. 핵심 기능 및 사용자 스토리 (User Stories)

(게임의 핵심 기능 요구사항은 플랫폼과 무관하게 동일합니다.)

| 기능 | 사용자 스토리 |
| :--- | :--- |
| **게임 보드** | 1. (As a Player) 나는 15x15 크기의 표준 오목판을 볼 수 있어야 한다.<br>2. (As a Player) 나는 게임판의 격자(교차점)가 명확하게 보여야 한다. |
| **돌 놓기** | 1. (As a Player) 나는 내 턴에 비어있는 교차점을 **클릭**하여 내 돌(흑 또는 백)을 놓을 수 있어야 한다.<br>2. (As a Player) 나는 이미 돌이 놓인 곳에는 돌을 놓을 수 없어야 한다. |
| **턴 관리** | 1. (As a Player) 나는 흑돌이 항상 첫 번째 수를 두는 것을 확인해야 한다.<br>2. (As a Player) 나는 현재 누구의 턴인지(흑/백) 명확하게 알 수 있어야 한다.<br>3. (As a Player) 한 번 돌을 놓으면 턴이 자동으로 상대방에게 넘어가야 한다. |
| **승리 조건** | 1. (As a Player) 내 돌(흑 또는 백)이 가로, 세로, 또는 대각선으로 **정확히 5개**가 연속으로 놓였을 때 게임이 즉시 승리로 끝나야 한다.<br> 2. (As a Player) 게임이 승리로 끝나면 "흑 승리\!" 또는 "백 승리\!"와 같은 명확한 승리 메시지를 볼 수 있어야 한다. |
| **게임 종료 및 재시작** | 1. (As a Player) 승리자가 결정되면 더 이상 돌을 놓을 수 없어야 한다.<br>2. (As a Player) 나는 **"새 게임" 버튼**을 클릭하여 언제든지 현재 게임을 리셋하고 새 게임을 시작할 수 있어야 한다. |
| **무승부 조건** | 1. (As a Player) 만약 게임판이 모두 찼는데도 승리자가 나오지 않으면, "무승부" 메시지를 볼 수 있어야 한다. |

### 3\. 게임 규칙 및 로직 상세

(게임 규칙은 플랫폼과 무관하게 동일합니다.)

  * **보드 크기:** 15x15 (225개의 교차점)
  * **시작:** 흑돌이 항상 먼저 시작합니다.
  * **승리 판정:**
      * 가로, 세로, 대각선 방향으로 자신의 돌이 **정확히 5개** 연속된 줄을 만드는 즉시 승리합니다.
      * "육목" (6개 이상)은 승리로 인정하지 않습니다.
      * "쌍삼" (3-3) 및 "쌍사" (4-4) 금지 룰은 **적용하지 않습니다.** (MVP 단순화 규칙)
  * **게임 데이터 구조 (C\#):**
      * 게임 보드 상태는 `int[15, 15]` 2차원 배열로 관리하는 것을 권장합니다.
          * `0` = 비어있음, `1` = 흑돌, `2` = 백돌

### 4\. UI/UX 고려 사항 (WPF)

  * **메인 윈도우 (`MainWindow.xaml`):**
      * 중앙에 게임 보드가 위치합니다. (WPF의 `Canvas` 컨트롤을 사용하여 격자 라인(`Line`)과 돌(`Ellipse`)을 그리는 것을 권장합니다.)
      * 상단 또는 측면에 현재 턴을 표시하는 `TextBlock` 또는 `Label`이 위치합니다. (예: "Current Turn: ⚫ (Black)")
      * 게임 상태(승리/무승부)를 알리는 `TextBlock` 또는 별도의 모달 `Window`를 사용합니다.
      * "새 게임 시작 (New Game)" `Button`이 명확하게 보여야 합니다.
  * **사용자 피드백:**
      * **Hover 효과:** 돌을 놓을 수 있는 교차점에 마우스를 올리면 (`Canvas`의 `MouseMove` 이벤트 활용), 해당 위치에 반투명한 돌 모양(`Ellipse`)을 표시하여 사용자에게 피드백을 줍니다.
      * **클릭 이벤트:** `Canvas`의 `MouseLeftButtonDown` 이벤트를 감지하여, 클릭된 좌표를 가장 가까운 교차점으로 계산한 후 `int[15, 15]` 배열을 업데이트하고 `Canvas`에 `Ellipse`를 그립니다.
      * **사운드 효과:** 돌을 놓을 때 `System.Media.SoundPlayer`를 사용하여 간단한 '착' 사운드를 재생합니다.
      * **승리 표시:** 승리 시, 승리한 5개의 돌을 연결하는 붉은색 `Line` 객체를 `Canvas`의 최상위에 추가하여 시각적으로 강조합니다.
  * **창 크기 조절 (Resizing):**
      * 메인 윈도우의 크기가 조절될 때, 게임 보드와 UI 요소들이 깨지지 않도록 `Viewbox` 컨트롤 내에 보드를 배치하거나, `Grid` 레이아웃의 비례 크기 조정을 활용하여 적절한 비율을 유지해야 합니다.
